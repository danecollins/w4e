As we created our server applications we found we had many scheduled maintenance tasks on different machines. One machine backs up the database every day while another machine sends out scheduled emails every hour, etc. As these tasks grew and spread over different servers and services we became concerned that if one of them failed we would not know.

We started having tasks sent notifications but that proved useless because what we cared about was what did not happen, not what did. This led us to create watch4.events, the service that tells us what did not happen.

How it Works

Watch4.events is based on tracking scheduled tasks. You create a 'monitor' for each task you want monitored. The 'monitor' which consists of a name and a frequency that a task occurs. Watch4.events then produces a URL which allow a task to check-in with the 'monitor'. You then make sure the task performs a get on that URL whenever it runs. If a task does not check-in on time you will receive an SMS with the task name.

At any time you can look at the check-in history of any of your tasks.
