Scheduled tasks are often spread across multiple machines which makes it difficult to monitor the status of the task processes. For example, one machine backs up the database, while another machine sends out scheduled emails, etc. With multiple scheduled tasks, manually monitoring the status of the scheduled tasks becomes overwhelming and makes it difficult to notice if one of the many tasks has stopped completely.

Sending a notification each time a task runs doesn't provide a proper solution for monitoring the health of an entire group of tasks. In regards to monitoring tasks, we mostly care about what did not happen, instead of what did happen. This led us to create Watch4Events, the service that tells us what did not happen.

How it Works

Watch4.events is based on tracking scheduled tasks. You can create a 'monitor' for each task that you want monitored. The 'monitor' consists of a name and a frequency that a task should occurs. Watch4.events then produces a URL which allow a task to check-in with the 'monitor'. You then make sure the task performs a get on that URL whenever it runs. If a task does not check-in on time you will receive an SMS with the task name.

At any time you can look at the check-in history of any of your tasks.
